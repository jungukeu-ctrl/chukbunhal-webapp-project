# 축분할 코칭 보드 - 고도화 PLAN

## 현재 상태
- 버전: 최종 완결판 (하이픈/물결 reps 범위 지원, Fallback 로직)
- 문제: 매 버튼 클릭마다 서버 호출 (느림), RPE/reps 메모 미활용, 추천 로직 단순

---

## 고도화 목표

### 1. 성능: 전체 데이터 1회 로드 아키텍처
- 앱 시작 시 `getAllData()` 1회만 호출
- 모든 루틴 데이터 + 전체 기록 + 대체운동 맵을 JS 메모리에 저장
- 프로그램/주/일 변경 → 서버 호출 없이 JS 필터링 (즉시)
- 종목 교체 → 서버 호출 없이 JS 재계산 (즉시)

### 2. 스마트 추천: 경험적 점진적 과부하 기반
**메모 파싱**
- 형식: `rpe7 reps3~6` / `rpe7 reps8-12` / `rpe7 reps8` + 뒤에 잡음 텍스트 가능
- 정규식: `/rpe(\d+(?:\.\d+)?)\s+reps(\d+)(?:[~-](\d+))?/i`

**같은 날짜 Top Set 선정**
- 같은 종목 + 같은 rpe + 같은 reps범위인 세트 중 최고 무게 선택

**매칭 우선순위 (P1 → P4)**
- P1: reps범위 일치 + RPE 일치 → 완전 일치
- P2: reps범위만 일치 → reps가 더 중요
- P3: RPE만 일치 → RPE만 맞는 경우
- P4: 둘 다 불일치 → 해당 종목 최근 기록 fallback

**추천 표시**
- 마지막 무게 그대로 표시 (자동 증가 없음, 사용자 판단)
- 매칭 기준 명시 (P1~P4 레이블)
- 최근 3세션 추이 표시

### 3. UI: 완료 체크
- 카드별 완료 체크 버튼
- 완료 시 시각적 구분 (초록 border)
- 헤더에 진행률 표시 (예: 3 / 6 완료)
